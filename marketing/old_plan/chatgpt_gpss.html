ChatGPT GPSs (general sketch)

GPSs here means GPT-powered systems: repeatable prompt workflows that speed up marketing execution (research, drafts, testing plans), while keeping humans responsible for truth, brand, and final decisions.

Rules of use (guardrails)

  • No secrets: don’t paste credentials, private user data, or non-public business-sensitive info.
  • Verify: treat outputs as drafts; validate claims, numbers, and competitor statements.
  • Consistency: maintain a single “source of truth” for messaging and product terminology.
  • Brand voice: the model adapts to a style guide; humans enforce it.

GPS library (what to build first)

GPS #1 — Niche discovery + ICP refinement

  • input: 5–10 candidate niches, current product strengths/limits
  • output: niche scorecard, hypotheses, top keywords, creator archetypes

GPS #2 — Positioning / message house generator

  • output: positioning statement, 3 message pillars, proof points needed, red-flag claims to avoid
  • keeps a “lexicon” of preferred terms (component/series/course/etc.)

GPS #3 — SEO topic map + internal linking plan

  • output: hub pages + supporting pages, suggested titles/meta, internal link graph
  • includes “thin-page detector” checklist

GPS #4 — Social content calendar + repurposing

  • output: 2–4 weeks of posts per channel, creator “launch kit”, repurposing matrix
  • aligns posts to landing pages (topic/creator/series)

GPS #5 — Partner / community outreach writer

  • output: personalized outreach drafts, follow-ups, and partner “offer” variants
  • strict: no invented facts; includes questions to fill before sending

GPS #6 — Email lifecycle drafts

  • output: welcome series, weekly digest template, creator onboarding + activation series
  • includes subject line variants + plain-text versions

GPS #7 — Experiment designer

  • output: hypotheses, success metrics, sample size heuristics (rough), test plan, risks
  • keeps focus on activation/retention, not vanity clicks

Prompt templates (copy/paste starting points)

Template A — Niche scorecard

Role: You are a growth strategist.
Context: Kvasir is a creator-first platform for structured learning content.
Task: Score the following niches for (1) search demand, (2) creator supply availability,
(3) community distribution paths, (4) content repeatability, (5) monetization potential.
Output: a table, top 3 niches, and the first 10 SEO page ideas for each.
Niches: [paste list]
Constraints: Don’t invent facts. Ask questions if required inputs are missing.

Template B — Message house

Role: Brand strategist.
Task: Draft a message house for Kvasir.
Include: positioning statement, 3 pillars, proof points we must gather, and phrases to avoid.
Tone: clear, not hype.
Product terms: [component/series/course/echo] (use consistently).
Constraints: If you’re unsure about product details, ask 5 clarifying questions first.

Template C — SEO topic map + linking

Role: SEO lead.
Task: Build a topic map for niche: [X].
Output:
- 1 hub page + 20 supporting pages with titles and intent
- Internal link plan (which pages link to which)
- Draft meta descriptions for hub + top 5 pages
Constraints:
- Avoid thin pages; each page must have unique value.
- No fabricated statistics; no competitor claims without sources.

Template D — Social calendar + repurposing

Role: Social editor.
Task: Create a 2-week content calendar for channels: [X, Y, Z].
Inputs: niche [X], core landing pages [paste URLs or page types], target audience [creators/learners].
Output: a calendar table + a repurposing matrix (one idea → multiple formats).
Constraints: Keep every post tied to a measurable CTA (follow/start/save/newsletter).

Template E — Outreach drafts (partners/community)

Role: Partnerships manager.
Task: Draft 3 outreach emails (short, medium, long) to: [partner name + context].
Offer: [curated page/webinar/creator bundle].
Inputs to request if missing: audience size, niche fit, tracking link.
Constraints: Don’t make up numbers or past results.

Operationalization (lightweight)

  • One shared doc: “Messaging + Lexicon” that every GPS references.
  • Prompt versions: store templates, note what changed and why.
  • Review loop: human review + publish; track results; refine the GPS.
