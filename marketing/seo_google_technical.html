4. Critical SEO Requirements for Indexing

⚠️ Must-Have for Google to Index

  1. 200 OK response: Pages must return HTTP 200 without auth redirects
  2. Crawlable content: Key content must be in initial HTML or rendered quickly by JS
  3. Unique titles + descriptions: Each page needs distinct <title> and <meta description>
  4. Canonical URLs: Set <link rel="canonical"> to prevent duplicate indexing
  5. Internal links: Pages must be reachable via HTML links from other indexed pages
  6. No robots block: Ensure robots.txt allows crawling and pages don't have noindex

6. JS Rendering Considerations

Option A (Recommended): Pre-render Key Pages

  • Why: Most reliable for Google indexing — HTML is immediately crawlable
  • Pages to pre-render: /deck, /contest?id=*, /legal?rules=*
  • How: Generate static HTML from API data during build/deploy

Option B: Dynamic JS with SEO Metadata

  • Current state: Pages load via JS after initial HTML shell
  • Improvement: Inject <title>, <meta>, and JSON-LD from JS immediately after data fetch
  • Risk: Google may not execute JS correctly → delayed or incomplete indexing

Option C: SSR via Lambda

  • How: Lambda endpoint returns full HTML with embedded content
  • When: Best for frequently-changing content like contest leaderboards
